# msrc_monitor

*Read this in other languages: [日本語](README.ja.md)*

A monitoring tool that fetches Microsoft's monthly Patch Tuesday primary data
(MSRC CVRF), summarizes it into facts, and notifies only on **changes** — then
helps generate a bilingual (Japanese / English) report from the frozen facts.

## Design philosophy

**"Change detection is automatic; meaning-making is manual."**

The program deliberately does **not** interpret or attribute. It aggregates
facts (CVE counts, severity, reboot class, product category, finder buckets)
and flags threshold-crossing *changes*. Turning those into a narrative — and
any causal or attributive claims — is left to a human.

In particular, the parser (`cvrf_parse.py`) never infers the identity of a
discoverer (e.g. "which AI/tool") from a credit string. It performs only
mechanical string classification. Roles are split so the data layer stays
free of judgment:

- **CISA KEV** = a discrete, edge-triggered notification signal.
- **FIRST EPSS** = a point-in-time reference value shown in the report only —
  never a notification trigger (it drifts daily).

## Data & privacy

**This repository intentionally ships no real MSRC data.**

Microsoft's CVRF acknowledgment credits contain third-party security
researchers' personal information (names, social handles, email addresses).
To respect that privacy, no real `state/` snapshots or CVRF fixtures are
included, and generated `state/*.json` is git-ignored so downstream users do
not accidentally commit researcher PII.

The test suite runs entirely on **synthetic data** generated by
`tests/fixtures/make_synthetic_cvrf.py` (dummy names only, `example.com`
addresses). To acquire real data, run the collector against the primary API
(below); the fetched data stays local.

## Getting started

```bash
# Python 3.12+, dependency: requests only
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Fetch the current month (writes to state/, which is git-ignored)
python collect.py
python collect.py 2026-Jul          # a specific month

# Run the tests (synthetic data; no network)
python tests/fixtures/make_synthetic_cvrf.py   # (re)generate the synthetic fixture
python -m pytest tests/ -q                      # or: python tests/test_regression.py

# Build the bilingual report drafts (requires Node.js + LibreOffice for PDF)
./report/build.sh
```

Notification and dead-man's-switch credentials (e.g. Pushover) are read from
environment variables / `env.sh` and are never committed (see `.gitignore`).

## Primary data source

- **MSRC CVRF API**: `https://api.msrc.microsoft.com/cvrf/v3.0/cvrf/{YYYY-MMM}`
  (e.g. `2026-Jul`). Reference implementation of the schema:
  `https://github.com/microsoft/MSRC-Microsoft-Security-Updates-API`.
- **Acknowledgments**: researcher credits are published by Microsoft in the
  Security Update Guide acknowledgments
  (`https://msrc.microsoft.com/update-guide/acknowledgement`). This project
  does not reproduce individual researcher identities.

## Repository layout

```
cvrf_parse.py        # self-contained CVRF parser + fact classification
collect.py           # fetch CVRF -> monthly fact summary (idempotent, freezes past months)
diff.py              # month-over-month thresholds + new-credit detection
draft.py             # facts-only draft generation (no interpretation)
notify.py            # edge-triggered notification
enrich.py            # KEV/EPSS enrichment layer (KEV=trigger, EPSS=reference)
report/              # bilingual report generation (docx/pdf) + anonymization gate
interpretation/      # human-written report interpretation (ja/en)
tests/               # regression + unit tests (run on synthetic data)
systemd/             # optional service + timer for scheduled runs
```
